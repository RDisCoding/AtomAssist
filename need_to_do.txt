PROJECT NAME: AtomAssist
TAGLINE: "Visual Customer Support for Faster Resolution."

======================================================================
PROJECT OBJECTIVE
======================================================================

We are building AtomAssist, a customer support platform for Atomberg that enables
real-time video-assisted troubleshooting between support agents and customers.

IMPORTANT:

Do NOT treat this as a video calling application.

Treat this as a customer support platform that uses video as one of its tools.

The final judging criteria are heavily focused on:

- Functionality
- Reliability
- Architecture
- User Experience
- Code Quality
- Operational Readiness

The architecture already exists and should NOT be rewritten.

Current Stack:

- Next.js Frontend
- Express Backend
- PostgreSQL + Prisma
- Socket.IO
- Mediasoup SFU
- JWT Authentication
- RBAC
- Session Management
- Chat Persistence

Focus only on:

1. UI/UX Improvements
2. Mobile Responsiveness
3. Bug Fixing
4. Reliability
5. Feature Completion
6. Demo Readiness

======================================================================
FINAL DEMO OBJECTIVE
======================================================================

The demo should tell a customer support story.

Scenario:

Customer reports:

"My ceiling fan is making a clicking noise."

Flow:

1. Agent logs in.
2. Agent creates support session.
3. Invite link generated.
4. Customer joins from mobile browser.
5. Video starts.
6. Customer shows the fan.
7. Agent diagnoses issue.
8. Customer sends image/PDF.
9. Chat updates in real time.
10. Admin dashboard tracks session.
11. Issue resolved.
12. Agent closes session.
13. Session marked CLOSED.
14. Session history remains accessible.

This should feel like a production-ready Atomberg support platform, not a generic video calling application.
